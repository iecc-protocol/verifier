# Security Policy 🛡️

## Supported Versions

We only provide security updates for the latest stable version of `@iecc/verifier`.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | ✅ Yes             |
| < 1.0.0 | ❌ No              |

## Reporting a Vulnerability

The IECC Consortium takes security seriously. If you find a security vulnerability in this protocol or library, please do not disclose it publicly.

Please report security vulnerabilities via email to: **security@iecc.world**

### What to include:
- A description of the vulnerability.
- A proof-of-concept (PoC) or steps to reproduce.
- Potential impact if exploited.

We will acknowledge your report within 48 hours and provide a timeline for a fix.

## Our Process

1. **Analysis**: We verify the vulnerability and its impact.
2. **Fix**: We develop and test a patch.
3. **Disclosure**: Once a fix is released, we will publish a security advisory to inform the community.

Thank you for helping keep the IECC ecosystem secure!
